# zerobench v0.0.1 Implementation Plan

> **For agentic workers:** Each task is self-contained and produces working, tested, committable software. Follow TDD: write the failing test → verify it fails → implement minimum to pass → verify pass → refactor if needed → commit. Use superpowers:subagent-driven-development to execute.

**Goal:** Ship zerobench v0.0.1 — an open-loop HTTP/1.1 benchmark tool with HDR-nanosecond latency measurement and a clean terminal report. Extensions (H2/H3/WS/SSE/Rhai/TUI) land as incremental features.

**Architecture:** Two-phase — Phase 1 compiles a `Plan` (from CLI / request-file / Rhai script); Phase 2 executes pure Rust (rate controller → dispatcher → transport → recorder → reporter). See `docs/design.md`.

**Tech Stack:** compio 0.18 (io_uring), hyper 1.4 (HTTP), cyper-core 0.8 (IO bridge), hdrhistogram 7 (stats), rustls 0.23 (TLS), ratatui 0.30 (TUI), rhai 1.19 (scripting).

### Task 2: Template system

**Files:**
- Create: `zerobench-core/src/template.rs`
- Create: `zerobench-core/src/rng.rs` — thin wrapper over `rand_xoshiro::Xoshiro256PlusPlus`
- Modify: `zerobench-core/Cargo.toml` — add `rand = "0.8"`, `rand_xoshiro = "0.6"`, `uuid = { version = "1", features = ["v7"] }`
- Test: `zerobench-core/tests/template.rs`

**Goal:** Implement §3 Template model — compile `{{...}}` strings to `Vec<Part>`, expand into `Vec<u8>` with zero per-call allocation beyond the output buffer.

**Supported variables (v0.0.1 vocabulary):**
```
{{uuid}} {{uuid4}} {{now_ms}} {{now_ns}} {{now_iso}}
{{counter}} {{counter_global}}
{{rand_int:MIN:MAX}} {{rand_hex:BYTES}} {{rand_str:LEN}}
{{env:NAME}} {{env:NAME:DEFAULT}}
{{line:FILE}}
{{var:NAME}}        — becomes Part::VarRef, resolved via VarRegistry
```

**Key interfaces:**
```rust
pub struct Template {
    parts: Vec<Part>,
    estimated_size: usize,
}

pub enum Part {
    Literal(Bytes),
    Uuid, Uuid4, NowMs, NowNs, NowIso,
    Counter(Rc<Cell<u64>>),
    CounterGlobal(Arc<AtomicU64>),
    RandInt { min: i64, max: i64 },
    RandHex { bytes: usize },
    RandStr { len: usize },
    Line(Arc<FixtureFile>),
    VarRef(VarSlot),
}

pub struct ExpandCtx<'a> {
    pub rng:             &'a mut Xoshiro256PlusPlus,
    pub counter:         &'a Rc<Cell<u64>>,
    pub scenario_vars:   &'a [Option<Bytes>],  // indexed by VarSlot
}

impl Template {
    pub fn compile(src: &str, vars: &mut VarRegistry) -> Result<Self, TemplateError>;
    pub fn expand_into(&self, out: &mut Vec<u8>, ctx: &mut ExpandCtx);
}
```

**Tests:**
- Literal-only template → `expand_into` copies bytes identically.
- `{{env:USER}}` with env set → baked at compile time as literal.
- `{{env:MISSING:fallback}}` → uses default.
- `{{uuid}}` produces 36-char hyphenated UUID.
- `{{rand_int:1:10}}` over 1000 expansions: every output in [1, 10].
- `{{now_ms}}` monotonically non-decreasing across expansions.
- `{{var:token}}` with empty slot → expands to empty bytes.
- Malformed template `{{unclosed` → `TemplateError::Syntax`.
- Unknown variable `{{bogus}}` → `TemplateError::UnknownVariable`.
- Nested braces `{{{{uuid}}}}` → literal `{{` + uuid + literal `}}`.
- Zero-alloc expansion verified via `#[test] fn no_alloc_after_warmup`.

**Done when:** tests pass, `cargo bench` (or a micro-benchmark) confirms expansion is ≤ 200ns for a 5-variable template.

**Commit:** `feat(core): Template compile + expand, {{...}} syntax`

**Goal:** Define the transport abstraction all HTTP/WS/SSE/H2/H3 impls satisfy.

**Key interfaces:**
```rust
pub struct Target {
    pub host: String,
    pub port: u16,
    pub tls:  bool,
    pub sni:  Option<String>,
}

pub struct TransportOpts {
    pub connect_timeout: Duration,
    pub request_timeout: Duration,
    pub max_conns:       usize,
    pub tcp_nodelay:     bool,       // default true
    pub insecure_tls:    bool,
}

pub struct Response {
    pub status:         u16,
    pub headers:        http::HeaderMap,
    pub body:           ResponseBody,
    pub bytes_sent:     u64,
    pub bytes_received: u64,
    pub ttfb:           Duration,
    pub total:          Duration,
}

pub enum ResponseBody {
    Buffered(Bytes),

pub trait Transport: Send + 'static {

        plan:    &RequestPlan,
        ctx:     &mut ScenarioContext,

}
```

### Task 10: Per-scenario RateScheduler (open-loop)

**Files:**
- Create: `zerobench-core/src/rate.rs`
- Modify: `zerobench-core/src/dispatcher.rs` — add `run_open_loop`
- Test: `zerobench-core/tests/rate_open_loop.rs`

**Goal:** Open-loop scheduler. Per scenario: emit tokens at configured rate into a shared MPMC channel. Worker pool pulls tokens and executes. Latency = `now - token.intended_start` (CO-free).

**Key interfaces:**
```rust
pub enum RateProfile {
    Constant(f64),
    Ramp { from: f64, to: f64, over: Duration },
    Stepped(Vec<(Duration, f64)>),
}

pub struct Token {
    pub scenario_id:    ScenarioId,
    pub intended_start: Instant,
}

pub struct RateScheduler { /* per-scenario state */ }

pub async fn run_open_loop<T: Transport>(
    plan:      &Plan,
    client:    T::Client,
    max_conns: usize,
    stop:      StopSignal,
) -> Vec<TaskStats>;
```

**Pool policy (from spec Q4):** start with `opts.max_conns` initial tasks. On token arrival with all workers busy and pool < `max_conns`: spawn additional worker up to cap. Past cap: drop token, increment `errors.keepup`.

**Tests:**
- Constant rate: target 1000 req/s for 3s against local server, assert actual within ±2%.
- Latency measured from intended_start: inject artificial 50ms server stall for 1s, assert p99 latency reflects it (CO-free check).
- Ramp 100→1000 over 2s: actual rate within ±5% at midpoint.
- Keepup: target 100k req/s against a slow server with max_conns=10, assert non-zero `errors.keepup`.

**Done when:** all rate tests pass; CLI supports `-r 1k` flag that picks open-loop path.

**Commit:** `feat(core): open-loop RateScheduler, keepup accounting`
